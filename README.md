# teen_social_log_analysis
청소년 타겟팅 소셜 서비스 행동 로그 분석 프로젝트
# 📉 3초 컷 이탈 막는 심폐소생술: 고객활성화 기반 재출시 전략 제안
> **익명 투표 SNS 데이터 분석을 통한 유저 리텐션 강화 및 서비스 개선 프로젝트**

## 1. 📝 Executive Summary (요약)
* **문제 상황**: 초기 핵심 경험 부재로 인한 높은 이탈률 (이탈 유저 비중 50.31%)
* **핵심 발견**: 학교 단위 유저 수가 450명 도달 시 활동성(평균 투표 수신 등)이 비약적으로 상승하는 '임계점' 발견
* **해결 전략**: 유저 수명(active_days) 예측 모델 기반의 타겟팅 관리 및 질문 콘텐츠 품질 관리 프레임워크 제안

## 2. 🎯 분석 배경 및 문제 정의
* **분석 배경**: 서비스 재출시를 앞두고 과거 실패 원인(Low Retention) 분석 필요
* **문제 정의**: 7GB 이상의 대용량 로그 데이터를 활용하여, 유저가 정착하지 못하고 떠나는 결정적 요인 파악
* **분석 목표**: 유저 활성화 임계점 도출 및 수명 예측 모델 구축을 통한 재출시 전략 수립

## 3. 🛠 분석 프레임워크 및 파이프라인
* **데이터 환경**: MySQL(데이터 저장), Google Colab / Python(분석 및 모델링)
* **파이프라인**: 
  * `raw_data` -> `preprocessing` (Hackle_Events 정합성 검증) -> `feature_engineering`
  * Airflow 기반의 대시보드 데이터 적재 파이프라인 설계 (`dashboard-pipeline.py`)

## 4. 🤖 모델링 및 해석 (유저 수명 예측)
* **목표**: 유저의 활동 수명(active_days_count)을 예측하여 이탈 위험군 조기 분류
* **방법론**: Random Forest 기반의 수명 예측 모델링 및 피처 중요도 분석
* **핵심 피처**: 포인트 사용 여부(consumption), 학교별 유저 밀도, 투표 상호작용 빈도 등

## 5. 🔍 핵심 발견 (Key Insights)
* **Insight 1**: 투표의 **상호작용(수신-발신)**이 형성될 때 사용자의 활동성이 가장 극대화됨.
* **Insight 2 (가장 중요!)**: 학교 내 유저 수가 **450명**을 넘어서는 순간, 평균 투표 수신 횟수가 **15회에서 168회**로 급증함.
* **Insight 3**: 단순 결제보다 **포인트 사용(소비 행동)**이 활동 수명과 더 강력하게 연결됨.

## 6. 💡 전략 제안 (Strategic Recommendations)
* **초기 정착 지원**: 가입 초기에 포인트 소비 경험을 유도하여 리텐션 제고
* **임계점 관리**: 학교별 유저 450명 달성을 최우선 목표로 하는 마케팅 및 운영 전략 시행
* **콘텐츠 관리**: '질문 단위' 품질 관리 프레임워크를 통한 지속적인 활성 피드백 루프 생성
